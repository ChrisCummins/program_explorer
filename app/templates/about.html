About

During the course of compiling a program, a compiler makes hundreds or even thousands of decisions about how to transform and optimize the code to give you the best performance. These decisions are made by heuristics which are painstakingly constructed and tuned by hand, but no heuristic is perfect. Machine learning offers the tremendous benefit of turning this tedious and error prone task into an automatic process in which heuristics are constructed by learning from past examples. In addition to being simpler to build, these machine learned heuristics often outperform hand tuned equivalents.

For a machine learning model to learn to reason about programs, it must be fed a suitable representation. Historically, representations of programs were constructed using ad-hoc analysis of programs to extract numeric feature vectors, but deciding which features to use is a laborious process which relies heavily on the time and expertise of the feature designer.

This site contains a tool to visualize ProGraML, a new graph-based representation of programs designed to capture both the attributes of a program's instructions, and the relations between them.

How to use Program Explorer

  1. On the program explorer page.
  2. Select the compiler IR type that you are using.
  3. Type, or copy, the compiler IR that you wish to visualize into the left pane.
  4. The compiler IR that you enter is sent to this server and processed into a ProgramGraph protocol buffer. The protocol buffer is shown in the central pane.
  5. The generated program graph is rendered into a Graphviz diagram on the right pane.
  6. To share or save your work, click the "Shareable URL" button. This generates a unique URL for this IR. Opening this link will restore the compiler IR and visualization.
